# DIKWP-VITA Mesh OS v1.0 发布说明

## 发布目标

把段玉聪已有的高密度 DIKWP、人工意识、可信智能体、开放基准、科研数字生命和文明场景资产组织为一个可下载、可访问、可运行、可复现、可批评、可修复、可镜像和可继承的开放数字生命主干。

## v1.0 核心能力

- 本地访问无条件增加本地生命史；
- 明示同意后发送每日轮换匿名 PulsePacket；
- GitHub Release 下载、Stars、Forks、仓库视图和克隆按增量进入公共生命账本；
- 受控镜像可使用无 IP/无 Cookie/无 User-Agent/无 Referrer 的最小化日志适配器，把每次访问或 ZIP 下载转换为聚合事件；
- `run-proof` 产生可验证运行证据；
- 反馈、反例、失败、修复、引用、采用、镜像心跳和非创始 Release 获得更高生命力权重；
- SQLite 追加式账本、事件去重、Ed25519 签名和 Merkle root 支持多节点联邦；
- 八维几何闭合避免访问量掩盖复现、治理和信任不足；
- EvolutionProposal 只提出变化，不自动批准高影响自我修改；
- 180 天连续性规则允许保留谱系的非冒充分叉。

## 重要科学边界

生命力分数是开放项目的组织代谢仪表，不是主观意识证明、科学正确性证书、个人价值评分或法律身份依据。下载和访问只能提供低权重能量；真正的公共生命力来自外部复现、反例、修复、采用和持续维护。

## 发布前必须确认

- 仓库正式名称和 URL；
- 段玉聪、贡献者、单位和版权信息；
- Apache-2.0 是否覆盖全部内容；
- Collector 运营者、允许 Origin、隐私公告和日志保留政策；
- 至少两名非创始 steward；
- Release asset、CI、Pages 和首次 run-proof；
- 对外传播不宣称软件已具有现象意识。
